# Business Rules

## 1. Общие правила

### 1.1. Заказ (Order)

- Заказ создаётся в статусе «Черновик».
- Заказ содержит одну или несколько позиций (Line-item).
- Каждая позиция соответствует одной единице оборудования.

### 1.2. Оборудование (Unit)

- Каждая единица оборудования может находиться только в одном из статусов:
  - Доступно
  - В резерве
  - В аренде
  - В ремонте

## 2. Резервирование

### 2.1. Создание резерва

- Резерв создаётся вручную сотрудником.
- Перед созданием резерва сотрудник обязан проверить:
  - корректность данных заказа
  - доступность оборудования

### 2.2. Эффект резерва

- При создании резерва:
  - статус оборудования меняется на «В резерве»
  - статус заказа остаётся «Черновик»

### 2.3. Ограничения

- Оборудование со статусом «В резерве» недоступно для других заказов
- Резерв может быть отменён:
  - вручную
  - автоматически (по истечению времени)

## 3. Отмена резерва

### 3.1. Условия отмены

- Резерв отменяется, если:
  - клиент не пришёл на склад
  - сотрудник отменил резерв вручную
  - истёк срок бронирования

### 3.2. Эффект отмены

- статус заказа → «Отменен»
- статус оборудования → «Доступно»

## 4. Выдача оборудования

### 4.1. Условия выдачи

- клиент должен прибыть на склад
- сотрудник (кладовщик) подтверждает выдачу

### 4.2. Эффект выдачи

- статус заказа → «Активен»
- статус оборудования → «В аренде»

## 5. Возврат оборудования

### 5.1. Общие правила

- Клиент обязан вернуть всё оборудование сразу
- Частичный возврат не допускается

### 5.2. Проверка оборудования

- Каждая единица оборудования проверяется отдельно
- Для каждой единицы определяется состояние:
  - исправно
  - повреждено

## 6. Обработка состояния оборудования

### 6.1. Исправное оборудование

- статус → «Доступно»
- депозит → возвращается клиенту

### 6.2. Поврежденное оборудование

- создаётся Repair Ticket
- статус → «В ремонте»
- депозит → удерживается

## 7. Финансовая логика (депозиты)

### 7.1. Депозит привязан к позиции

- Депозит рассчитывается и учитывается на уровне Line-item

### 7.2. Возврат депозита

- Возвращается только для оборудования со статусом «Доступно»

### 7.3. Удержание депозита

- Удерживается для оборудования со статусом «В ремонте»

## 8. Завершение заказа

### 8.1. Полное закрытие (Closed)

- Заказ получает статус «Закрыт», если:
  - всё оборудование возвращено
  - все единицы оборудования исправны

### 8.2. Частичное закрытие (PartiallyClosed)

- Заказ получает статус «Частично закрыт», если:
  - всё оборудование возвращено
  - хотя бы одна единица оборудования повреждена

## 9. Инварианты системы

- Оборудование не может одновременно находиться в нескольких статусах
- Заказ не может стать «Активен» без выдачи оборудования
- Заказ не может быть «Закрыт», если есть оборудование в статусе «В ремонте»
- После возврата все единицы оборудования должны иметь статус:
  - либо «Доступно»
  - либо «В ремонте»
